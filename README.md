## 🚀 Projets phares

### 🐱🐶 **Dogs vs Cats — Benchmark CNN vs Régression Logistique** *(déployé)*
> *Étude expérimentale isolant l'apport du biais inductif convolutif — PyTorch Lightning + démo live*
- 🔬 **Comparaison rigoureuse** : régression logistique sur pixels bruts (**56.4%**) vs CNN custom 4 blocs (**88.6%** accuracy) — données, loss et trainer **strictement identiques**, seule l'architecture varie
- 🧱 **Package Python modulaire** (`src/`) : DataModule, modèles, training, évaluation, inférence — configs **YAML avec héritage**, zéro hyperparamètre en dur
- ⚡ **PyTorch Lightning** : `BaseClassifier` partagé garantissant une comparaison symétrique, callbacks (EarlyStopping, ModelCheckpoint, LR Monitor), scheduler cosine
- 🧪 **14 tests unitaires** (pytest), **CI GitHub Actions** (ruff + black + pytest sur Python 3.10/3.11), **Dockerfile** multi-stage
- 🎮 **Démo live déployée** sur **Hugging Face Spaces** : comparaison des 2 modèles côte-à-côte sur n'importe quelle image uploadée
- 📊 Suite d'évaluation complète : matrices de confusion, courbes ROC, distributions de probabilités, galerie d'erreurs qualitatives

🔗 [**Code source**](https://github.com/Yobapatrick/dogs-vs-cats) · [**Démo live 🤗**](https://huggingface.co/spaces/PatrickYoba/dogs-vs-cats)

### 🦺 **SafetyVision — Monitoring temps réel de la conformité EPI sur chantier** *(en cours)*
> *Computer Vision + MLOps complet, déployé en production*
- 📦 Constitution et exploration d'un **dataset de 2 801 images au format YOLO** (casques, gilets, masques, ouvriers, engins)
- 🎯 **Fine-tuning de YOLO26** (Ultralytics, janvier 2026) sur GPU avec architecture **end-to-end NMS-free**
- 🔁 Logique métier de **tracking multi-objets** via **ByteTrack** (Transformer-based) pour un comptage dédupliqué des ouvriers
- ⚡ Backend **FastAPI** (endpoints **REST + WebSocket**) avec persistance **PostgreSQL** des sessions
- 🐳 Conteneurisation **Docker multi-stage** et déploiement sur **Google Cloud Run** (GCP), **CI/CD via GitHub Actions**
- 📊 Dashboard analytics **Streamlit** : upload vidéo, métriques live (taux de conformité, alertes temps réel)

### 🧠 **Fine-Tuned LLM Deployment — Génération de CV enrichis sans hallucination**
> *Fine-tuning supervisé (SFT) avec **réduction mémoire de 65%** via QLoRA*
- 📂 Constitution et structuration d'un **dataset JSONL de 2 504 lignes** (description candidat → CV JSON cible)
- 🔧 Pipeline d'entraînement automatisé : **tokenisation, split train/eval 80/20, templating chat**
- ⚙️ **Fine-tuning supervisé (SFT)** du Transformer **Phi-3 Mini 4K Instruct** (Microsoft, **3.8B paramètres**) via **QLoRA 4-bit** (NF4 + double quantization), librairies **Hugging Face Transformers / PEFT / TRL**, **4 epochs en bf16** sur GPU Colab
- 📋 **Schéma de sortie strict** imposé par system prompt (informations, résumé, compétences, expériences, éducation…)
- 🚀 Déploiement via **API Flask (latence 125 ms)** exposée publiquement par **Ngrok**, testée sur batch de candidats

### 🔄 **Système RAG Correctif (CRAG) — Question Answering avec auto-évaluation et fallback web**
> *Architecture RAG nouvelle génération avec auto-correction et orchestration LangGraph*
- 📚 **Ingestion d'une base documentaire multi-PDF** (4 documents, **2 150 pages → 5 000 chunks** via `RecursiveCharacterTextSplitter`, `chunk_size=1200`, `overlap=100`)
- 🧮 Vectorisation via **embeddings OpenAI** et indexation dans une base vectorielle **FAISS** pour la recherche sémantique
- 🧩 Pipeline RAG correctif orchestré avec **LangGraph** : `retrieve → grade documents` (évaluateur LLM, sortie structurée **Pydantic**, score binaire) → `transform query → web search fallback` (**Tavily**) → `generate`
- 🤖 LLM de génération et d'évaluation : **GPT-4.1-nano** via **LangChain** (Transformer décodeur), avec prompts dédiés par nœud
- 🔁 **Logique conditionnelle** : si documents jugés non pertinents, **reformulation + recherche web automatiques** avant génération
